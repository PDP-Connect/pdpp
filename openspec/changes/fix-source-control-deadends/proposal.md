## Why

Live source-control surfaces can send the owner into dead ends or stale conclusions:

- Browser-surface assistance can mint a streaming session even when the companion cannot attach to the resolved browser target.
- Source list/detail surfaces can render stale summary evidence after run and record changes mark the summary projection dirty.
- Resting per-stream gaps can read like active "checking" work, which obscures whether the owner can act.

## What Changes

- Preserve route-resolved browser-surface targets through the default streaming companion factory.
- Pin a regression test for no-response browser assistance using the production factory path, not only a mock factory.
- Require source-control reads to reconcile dirty maintained summary evidence before presenting it.
- Clarify owner-facing source copy so unavailable counts and unresolved coverage do not imply active work.

## Capabilities

Modified:

- `reference-run-assistance`
- `reference-connection-health`

## Impact

- Owner repair pages stop pointing at unusable browser streams for active browser-session repair.
- Source health reads are based on current canonical evidence rather than stale dirty rows.
- No protocol or grant-scope change.
