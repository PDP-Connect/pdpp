## 3. Timer lifecycle

- [x] Own the `setInterval`/`unref()`/start/stop contract in a dedicated, independently testable module: `runtime/browser-surface-lease-sweep-timer.ts` (`createBrowserSurfaceLeaseSweepTimer`), with injectable `setIntervalFn`/`clearIntervalFn`.
- [x] Interval length reads an optional env override (`PDPP_NEKO_LEASE_SWEEP_INTERVAL_MS`, same shape as existing `browser-surface-leases.ts` env knobs); default `DEFAULT_NEKO_LEASE_SWEEP_INTERVAL_MS` derived from the existing lease-wait-timeout scale.
- [x] Expose `stopBrowserSurfaceLeaseSweep` and call it from the CLI shutdown path alongside the scheduler manager and delivery worker (defense-in-depth, fires fastest on a genuine process shutdown).
- [x] **Stop only when ALL bound close sources have closed, not the first.** `timer.stopWhenAllClosed(sources)` — a method ON the timer object, renamed from an earlier `stopOnCloseOf` design that incorrectly stopped on the first close — tracks a countdown across every given source and calls the timer's private `stop` only once every source has fired its own `'close'` event. The sweep's `sweep()` closure is reachable through EITHER `asServer` or `rsServer` (neither exclusively owns the controller's lifetime); closing one while the other remains open must NOT stop the sweep. `stopWhenAllClosed` lives on the timer, not as a standalone `server/index.js` function (an earlier draft did that, then left the function dead after the call site moved) — keeping the binding testable without importing the ~150-dependency `server/index.js` module, and keeping "who can stop this timer, and under what condition" co-located with the rest of the timer's lifecycle contract.
- [x] **Construct early (unstarted); arm (bind + start) only after the entire fallible boot succeeds.** `createBrowserSurfaceLeaseSweepTimerFor(controller, browserSurfaceControllerOptions, logger)` constructs the timer without calling `.start()`, called right after the controller exists. `armBrowserSurfaceLeaseSweepAfterBoot(timer, browserSurfaceControllerOptions, asServer, rsServer)` is the ONLY call site of `.start()`; it calls `stopWhenAllClosed([asServer, rsServer])` first, then `.start()` — bind before start. `startServer` calls `armBrowserSurfaceLeaseSweepAfterBoot` as the LAST statement before its `return`, after every other fallible `await` in that function (buildAsApp/asApp.listen/buildRsApp/rsApp.listen/schedulerManager.start/auto-enroll/the optional startup-backfill await) has already succeeded. A boot failure anywhere before that line means the timer was constructed but never started — a real, previously-possible leak (start-before-bind, started long before the fallible remainder of boot), not a residual. Both functions are exported so this EXACT production sequencing is directly testable, not a reimplementation of it.
