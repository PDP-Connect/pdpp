# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities in `@opendatalabs/remote-surface` to
`security@vana.org`. The same contact covers both the substrate package and
the reference implementation that consumes it.

When reporting, include:

- a description of the vulnerability and its potential impact;
- minimal reproduction steps or a proof-of-concept;
- the package version (or commit SHA, if reporting from source);
- whether the issue is already public.

Please do not file public GitHub issues for suspected vulnerabilities until
the maintainers have had a chance to triage and respond. Coordinated
disclosure timelines will be agreed with the reporter.

## Supported Versions

`@opendatalabs/remote-surface` is currently in pre-release (`0.0.x`, `private: true`).
Public version support windows will be documented here when the first public
release is cut.
