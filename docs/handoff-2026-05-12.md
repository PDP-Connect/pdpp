## 4. Architectural anti-patterns to avoid

Each of these burned cycles this session. Don't repeat them.

1. **Wrapper rewrite under the guise of bug fix.** One agent expanded a
   "fix tap-to-click" task into a 702-line stream-viewer.tsx diff. We
   reverted. Stay surgical; surface scope expansion as a question, not a
   commit.

2. **Validation without observable telemetry.** Multiple agents tried to
   validate a fix by tapping the phone, observing no error, and declaring
   success. Doesn't work. Telemetry must show the new code path firing,
   or the new behavior must be visible on screen and screenshot-able.

3. **Compose-file invocation drift.** Twice this session, services were
   brought up without the neko overlay; env vars silently dropped;
   debug "why isn't this working" loops followed. Always both compose
   files, always `--env-file .env.docker`. Track #41 has the doc fix.

4. **Skipping the canonical-source check.** When something is broken,
   verify against three sources before coding a fix. (Step 2's agent did
   this exemplarily — found that `bindTouchHandler` + `buttonDown/Up`
   double-emit causes Brave double-click; chose mouse-only path. Use as
   template.)

5. **Subagent over-trust.** Subagents are useful for parallel work and
   token efficiency, but: they get stuck in build-poll loops; they
   sometimes ignore anti-requirements and rewrite things they shouldn't;
   they ballooned scope twice this session. Set hard timeouts (≤25 min
   typical), strict anti-requirements, and require concrete validation
   evidence in their reports. Read their diffs before accepting their
   work — `git diff --stat` is your friend.

6. **Patching the React handler stack.** The expert was emphatic: the
   React event handler surface is NOT the foundation. Every behavior
   fix should live in the adapter/controller layer. Resist any urge
   to add another conditional to `<NekoSurface>`.

7. **"Just kill that process."** Twice this session the user had a
   dev server on a port we needed. Ask before killing user-owned
   processes; offer to kill if user confirms.
